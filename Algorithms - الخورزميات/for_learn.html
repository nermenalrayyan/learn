تقوم الخوارزميات بتبسيط المشكلة و وضع خطوات لحلها بشكل متسلسل ومنطقي باللغة البشرية وليس البرمجية ثم بعد الانتهاء من إجاد الحل المناسب نقوم بتحويلها للغة برمجية ليفهمها الحاسوب

المصدر

تعلم التفكير المنطقي و كيفية حل المشكلات أهم من تعلم لغات البرمجة تعلم البرمجة وليس لغة البرمجة

ما هي الخوارزميات

عند التفكير بشئ برمجي ان كانت مشكلة او ميزة لا تكتبه مباشرا على الكود بل هنالك عدة خطوات يجب اتباعها قبل تكويد الفكرة
احدى الخطوات هي التفكير في حل المشكلة او الميزة وكتابتها بشكل منفصل ومرتب بالطريقة الخوارزمية

الخوارزميات هي مجموعة خطوات رياضية و منطقية ومتسلسلة لحل مشكلة ما

شروطها

تكون كتابة الخطوات بصورة مفصلة و واضحة

الترتيب في كتابة الخطوات

مثال: اذا احتجنا لعمل كيك
اولا: توفير المكونات
ثانيا: خلط الكيك مع بعضها
ثالثا: خبز الكيك

التراكيب الاساسية في الخوارزميات:

1. التركيب التسلسلي ( Sequential Struction )
التركيب التسلسلي لا يتكرر

2. تركيب الاختيار ( Selection Structure )
يتعامل اسلوب if

3. تركيب التكرار ( Repetition Structure )
يقوم بعملية التكرار

لماذا نتعلم الخوارزميات

1. توثيق العمل وحفظه من النسيان

2. معرفة بداية ونهاية البرنامج

3. سهولة تطوير البرنامج من قبل المبرمجين الاخرين للمقدرة على قراءتهم الخوارزميات المشاكل التي واجهتها وطرق الحلول التي استخدمتها في حل هذا المشاكل

4. القدرة على اختيار الحل الامثل وذلك عن طريق كتابة عدة خوارزميات و اختيار الافضل و الانسب بينهم لحل هذا المشكلة

تعلم الخوارزميات الدرس الثاني

المصدر

يتم كتابة الخوارزميات بطريقتين وهم

الطريقة الاولى اسمها Flow Chart ( المخطط الانسياب او خرائط التدفق )

Flow Chart نفس طريقة Pseudo Code ولكن بطريقة أشكال وتسمى بال عناصر مخطط الانسياب او عناصر خرائط التدفق لكل شكل له أستخدام بمعنى

1. الشكل البيضاوي نستخدمه لبداية ونهاية البرنامج

2. الاسهم تستخدم لربط بين الاشكال

3. متوازي الاضلاع تستخدم للإدخال و الاخراج ( الادخال يعني الاستقبال و الاخراج يعني طبع

4. المستطيل تستخدم لعمليات الحسابية

5. شكل المعين تستخدم لإتخاذ القرار و الاختيار

أنواع Flow Chart

1. خرائط التتابع البسيط

2. خرائط ذات فروع

3. خرائط الدوران البسيط

4. خرائط الدوران المعقد


صورة توضع كتابة خوارزمية لبرنامج يقارن بين ثلاث ارقام ويطبع اكبرهم
ملاحظة: علامة ( < ) تعني أكبر من و علامة ( > ) تعني أصغر من

الطريقة الثانية أسمها Pseudo Code ( الشفرة الزائفة )

تكون الخطوات كالتالي

1. بداية البرنامج

3. احسب x + y = z

4. أطبع z

5. نهاية البرنامج

تعلم الخوارزميات الدرس الرابع ( خوارزمية التكرار)

المصدر

أكتب خوارزمية لبرنامج يقوم بطباعة مجموع الاعداد من 1 الى 3 منطق هذه الخوارزمية كالتالي:

1. اول سطر هو البداية

2. قراءة متغيرين x, sum
متغير x عداد الارقام
متغير sum يقوم بتخزين النتيجة النهائية

3. x = 1 sum = 0
وضع 1 لل x لان بداية العد يبدء من 1
وضع 0 لل sum لانه لا يوجد حاليا عملية حسابية اذا اصبح هنالك عملية حسابية يبدء sum بإظهار الناتج

4. x + sum = sum
بمعنى 1 + 0 = 1

5. x + 1 = x
بمعنى 1 + 1 = 2

6. x > 3
تسمى هذه الخطوة بالشرط وتعني يتم عمل هذه الخطوة لوضع حد للبرنامج بعنى اذا اصبح x اكبر من 3 فيجب توقيف العملية وعدم تكرارها
فاذا كانت العمليات خطأ وال x ليست اكبر من 3 يجب ارجاع عمل العمليات الحسابية حتى يصبح x اكبر من 3 وبعدها تتوقف العملية الحسابية

7. أطبع sum

8. النهاية
